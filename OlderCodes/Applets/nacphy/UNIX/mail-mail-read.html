§ 12.C.2: Reading Mail with mail

Reading your mail with the mail utility is easy:

    > mail
    No mail
As usual, we have no mail.

If you do have mail, the mail utility displays a one line entry: 

     
"/var/spool/mail/john": 2 messages 2 new
>N  1 goldie             Tue Nov 12 10:37  11/301  "meow"
 N  2 boru               Tue Nov 12 10:37  12/300  "woof"
Message #m:
Here are some responses for you to try:
Control Commands:
? List all subcommands
q Quit - apply mailbox commands entered this session
xQuit - restore mailbox to original state
! [cmd] Start a shell, run [cmd], and return to mailbox
cd [dir] Change directory to [dir] or $HOME
Display Commands:
t [msg-list] Display messages in [msg_list] or current message
nDisplay next message
f [msg-list] Display headings of messages
h [num] Display headings of group containing message [num]
Message Handling:
e [num] Edit message [num] (default editor is ex)
d [msg-list] Delete messages in [msg-list] or current message
u [msg_list] Recall deleted messages
s [msg-list] Append messages (with headings) to
w [msg-list] Append messages (text only) to
pre [msg-list] Keep messages in system mailbox
Creating New Mail:
m [addrlist] Create/send new message to addresses in
r [msg-list] Send reply to senders and recipients of messages
R [msg-list] Send reply only to senders of messages
aDisplay list of aliases and their addresses
